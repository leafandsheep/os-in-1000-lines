但是我們還要開始放入kernel，下一步從linkerscript開始

kernel.ld:用來指定核心在記憶體的位置，也去分配指定的函式和變數位置
指令如下：
ENTRY(boot)

SECTIONS {
    . = 0x80200000;

    .text :{
        KEEP(*(.text.boot));
        *(.text .text.*);
    }

    .rodata : ALIGN(4) {
        *(.rodata .rodata.*);
    }

    .data : ALIGN(4) {
        *(.data .data.*);
    }

    .bss : ALIGN(4) {
        __bss = .;
        *(.bss .bss.* .sbss .sbss.*);
        __bss_end = .;
    }

    . = ALIGN(4);
    . += 128 * 1024; /* 128KB */
    __stack_top = .;
}

我從中學到的幾個結構要素有 在一開始SBI結束後 透過.直接指定接下來的要執行的記憶體位置也就是kernel所需的結構的記憶體位置，也有學到* 蒐集相關檔名的技巧，還有最後留個stack給基礎的函式呼叫留空間，阿對還有bss的處領0的技巧
